1번 - 채팅방에 접속했을 때

* 사용자가 채팅 페이지에 접속하면 사용자를 구별하는 socket.id를 "user connected"라는 말과 함꼐 서버 콘솔에 찍어준다.
  이 id는 자동으로 생성되는 영어와 숫자로 만들어진 문자열로, 채팅방에서 사용할 닉네임과는 다르다.

* 처음 접속했을 때는 익명1, 익명2와 같은 순서로 닉네임이 생성된다.
  따라서 이 변수 name에 값을 넣어준다.

* 또 닉네임이 변경될 떄 비교해주기 위한 값이 필요하기 때문에, socket.name에도 현재 생성된 닉네임 값을 넣어준다.

* id.to(socket.id).emit: 서버가 해당 socket.id에만 이벤트를 전달한다. 채팅방에 접속한 유저마다 닉네임이 다른데,
  모두 바뀌지 않고 각 유저에게만 닉네임이 보여지기 위해 io.to(socket.id).emit을 이용하여 create name이라는 이벤트를 발생시킨다.


2번 - 접속이 끊겼을 때

* 접속한 사용자가 채팅방을 나가면, "user disconnected"라는 말과 함꼐 socket.id와 socket.name을 서버 콘솔에 찍어준다.


3번 - 메시지를 보냈을 때

* 사용자가 전송 버튼을 눌러서 메시지를 보냈을 경우, 채팅방에 출력해줄 메시지를 msg 변수에 넣어준다. 여기서는 닉네임과 메시지를 합쳐준다.

* 사용자가 닉네임을 변경했을 떄를 위해 socket.name에 전달받은 name을 넣어준다.

* 서버 콘솔에 메시지를 출력해주고, receive message라는 이벤트를 발생시키고 메시지를 전달해준다.
